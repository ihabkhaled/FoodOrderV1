# Incident management

Declare severity by user impact, security/privacy/data integrity, and duration. First actions are stop harmful changes, preserve minimal evidence, assign incident lead, establish communication, mitigate safely, and validate recovery. Postmortems are blameless and link corrective actions to owners, dates, tests, rules, runbooks, or architecture changes.
