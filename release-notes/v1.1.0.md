# FoodOrderV1 v1.1.0

A UX-focused release: a redesigned responsive shell, a prompt-density versioning system, and raised
platform floors — all shipped as small, verified, pushed commits.

## Highlights

- **Responsive UI overhaul.** Desktop now uses a persistent left **sidebar**; mobile keeps a slim top
  bar and a bottom nav with correct safe-area padding so content never hides behind it. Fixed the
  glued stat values (`0Buckets` → **0** over **Buckets**), moved toasts to a bottom, non-overlapping
  position, and corrected a stepper that rendered vertically. Consistent spacing, focus rings, hover
  states, and dialog backdrop throughout. Verified with a new Playwright UI gate.
- **Prompt-density versioning.** SemVer with one place-of-record; the bump level is chosen by how large
  the driving prompt is (localized → patch, feature → minor, breaking → major). New
  `skills/versioning`, `rules/versioning.md`, `docs/operations/versioning.md`, `CHANGELOG.md`, and a
  single-owner `tools/release/bump-version.mjs` that syncs package.json + Android versionName/versionCode.
- **Raised platform floors.** Node **>= 26**, Android **minSdk 29 (Android 10)**, iOS deployment target
  **14.0**. (iOS 13 is not supported by Capacitor 8; documented in `docs/operations/platform-support.md`.)
- **Repo hygiene.** Full worktree + history scan: no secrets, keystores, `.env`, APKs, build outputs,
  junk, or orphan files tracked; the production Firebase key was never committed.
