# FoodOrderV1 v1.1.1

Patch release (low-density): build hygiene and e2e reliability.

## Highlights

- **No sourcemaps in the production bundle.** Vite `build.sourcemap` is now `false`. Sourcemaps were
  copied verbatim into the native asset payload — the primary win is **no longer shipping readable
  source to the device**; the debug APK also shrinks (12.2 MB → 10.8 MB, and web `dist/` 5 MB → 1 MB).
  Dev builds (`npm run dev`) keep sourcemaps.
- **Reliable e2e gate.** Local Playwright now retries once (CI keeps two) and uses a 15 s action
  timeout, absorbing cold-start flakes when workers boot in parallel against a freshly built preview.
