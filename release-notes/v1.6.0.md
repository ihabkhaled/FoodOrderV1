# FoodOrderV1 v1.6.0

Architecture release: the entire application source is migrated to a strict,
mechanically enforced module-first structure. User-visible behavior is
unchanged (all Playwright journeys pass unmodified); this release hardens the
codebase itself.

## Highlights

- Module-first source layout: `src/app` (composition), `src/modules/*`
  (auth, buckets, group-orders, orders, social, notifications, dashboard,
  settings, session, data-access), `src/shared`, `src/platform`,
  `src/packages` — with a one-way dependency direction.
- Project-owned ESLint architecture plugin (nine AST rules, 54 RuleTester
  cases) enforcing package ownership, hook isolation, public module surfaces,
  layer direction, browser/env centralization, route-path ownership, naming,
  and the enum ban at error severity.
- Every third-party dependency has one owning facade under `src/packages/`
  (firebase, router, icons, capacitor plugins, virtuoso), registered in a
  machine-readable ownership registry.
- Every screen is a thin container over a dedicated view-model hook;
  components are hook-free and receive prepared props.
- Runtime capabilities (environment, browser, device, network, storage) are
  centralized in `src/platform` adapters.
- Governance system: canonical `AGENTS.md`, 22 rule documents, 15 task
  skills, reviewer personas, context maps, ADRs 0001-0008, migration
  inventory, and a CI-gated agent-docs sync check (`quality:agent-docs`).
- Stricter TypeScript (`noImplicitReturns`, `noPropertyAccessFromIndexSignature`,
  `verbatimModuleSyntax`, no unreachable code/labels) and enforced coverage
  ratchets (pure shared helpers at 100%).
- Localized the last hardcoded copy (not-found screen, error-state retry) in
  both English and Arabic.
