# FoodOrderV1 v1.6.0

Architecture and experience release: the application source is migrated to a strict,
mechanically enforced module-first structure, while the shared UI receives a substantial
responsive polish pass across PWA, mobile, tablet, and desktop web layouts.

## Highlights

- Module-first source layout: `src/app` (composition), `src/modules/*`
  (auth, buckets, group-orders, orders, social, notifications, dashboard,
  settings, session, data-access), `src/shared`, `src/platform`,
  `src/packages` — with a one-way dependency direction.
- Project-owned ESLint architecture plugin (nine AST rules, 54 RuleTester
  cases) enforcing package ownership, hook isolation, public module surfaces,
  layer direction, browser/env centralization, route-path ownership, naming,
  and the enum ban at error severity.
- Every third-party dependency has one owning facade under `src/packages/`
  (firebase, router, icons, capacitor plugins, virtuoso), registered in a
  machine-readable ownership registry.
- Every screen is a thin container over a dedicated view-model hook;
  components are hook-free and receive prepared props.
- Runtime capabilities (environment, browser, device, network, storage) are
  centralized in `src/platform` adapters.
- Governance system: canonical `AGENTS.md`, 22 rule documents, 15 task
  skills, reviewer personas, context maps, ADRs 0001-0008, migration
  inventory, and a CI-gated agent-docs sync check (`quality:agent-docs`).
- Stricter TypeScript (`noImplicitReturns`, `noPropertyAccessFromIndexSignature`,
  `verbatimModuleSyntax`, no unreachable code/labels) and enforced coverage
  ratchets (pure shared helpers at 100%).
- Localized the last hardcoded copy (not-found screen, error-state retry) in
  both English and Arabic.

## UI and responsive experience

- Richer visual depth for cards, navigation, forms, dialogs, toasts, statuses,
  bucket actions, social cards, and responsive shells without changing domain behavior.
- Cooler primary, secondary, success, danger, and icon-button states with clearer focus,
  hover, pressed, disabled, and touch behavior.
- Improved PWA/mobile bottom navigation, tablet spacing, desktop sidebar styling,
  short-height landscape handling, safe-area spacing, long-copy wrapping, and reduced-motion
  fallbacks.
- New accessible loading presentation with a live status, progress icon, and skeleton shimmer.
- Friend-group member counts now render as visually separate metadata instead of sticking to
  the group title, while edit/delete controls remain visible and comfortably tappable.
- Notification overlays retain viewport containment, readable content, scrolling, RTL-safe
  positioning, and correct stacking above routed content.

## Validation and test expansion

- Vitest/Testing Library coverage now includes the shared loading state and local notification
  ordering, subscription, read-state, cap, and malformed-data behavior.
- Primary Playwright gate covers Chromium desktop, Pixel 7 Chrome, and iPad Mini Chromium.
- A dedicated cross-browser workflow covers desktop Firefox, desktop WebKit, and iPhone 15 Pro
  Safari.
- Responsive journeys cover portrait, landscape, tablet, desktop, overlays, long copy,
  touch targets, group title/member metadata separation, action visibility, RTL-safe layout,
  and horizontal overflow.

## Known limitations

- iOS native project, entitlement, signing, and App Store validation still require a macOS
  environment (documented in `docs/migration/native-security-audit.md`). Mobile Safari
  Playwright validates PWA/web rendering only.
- Cross-feature persistence gateways remain in `modules/data-access`
  (EXC-1); per-feature splitting awaits characterization tests.
