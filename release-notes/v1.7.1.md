# FoodOrderV1 v1.7.1

v1.7.1 turns Gama3 Orderak into a multilingual, SEO-ready, crawler-friendly product: the
public marketing surface is prerendered in twelve locales, every application catalog is
fully translated, social sharing renders branded previews, and AdSense readiness is
enforced through route-level eligibility rules — all without changing any ordering or
collaboration workflow.

## Highlights

### Multilingual expansion (12 locales)

- Supported locales: `en`, `ar`, `it`, `fa`, `fr`, `de`, `es`, `pt-BR`, `hi`, `th`,
  `zh-CN`, `ja`. Arabic and Persian render RTL; all others LTR.
- Every user-facing catalog (core, settings, group orders, order sessions,
  session invites, social) ships genuine translations for all twelve locales — no
  English fallbacks, no aliased catalogs.
- Firebase error copy is a complete 12-locale catalog behind the packages facade.
- Browser/device language is detected on first visit; the explicit selection persists
  via Capacitor Preferences and the signed-in profile. `lang`/`dir` follow the locale.
- `npm run i18n:check` is the deterministic gate: exact key parity, interpolation-token
  parity, blank detection, unknown keys, and untranslated-clone detection for every
  catalog plus structural completeness of public marketing content.

### Language dropdown

- The pressable language toggle is replaced by an accessible native `<select>` showing
  each language in its own script (English, العربية, Italiano, فارسی, Français, Deutsch,
  Español, Português, हिन्दी, ไทย, 简体中文, 日本語).
- Available in the public header, auth screen, app shell, session-invite screen, and
  settings; keyboard- and screen-reader-friendly; correct in narrow and RTL layouts.

### SEO, prerendering, and social sharing

- Public routes (`/`, `/about`, `/how-it-works`, `/features`, `/group-ordering`,
  `/split-bill-and-receipts`, `/faq`, `/contact`, `/privacy`, `/terms`) are prerendered
  at build time for every locale with unique titles, meta descriptions, one `<h1>`,
  canonical URLs, `hreflang` alternates + `x-default`, Open Graph and Twitter Card
  metadata, and JSON-LD structured data.
- `robots.txt` and localized `sitemap.xml` are generated; only the canonical production
  origin is indexable — previews and non-production deployments send `noindex`.
- Authenticated application routes stay out of the sitemap and carry
  `X-Robots-Tag: noindex` plus private cache headers.
- Branded 1200×630 social preview image plus full favicon/PWA icon set.

### AdSense-safe content architecture

- Route-level ad-eligibility registry: only content-rich public editorial pages are
  eligible; auth, dashboard, settings, forms, receipts, loading/error/empty states, and
  every unknown route default to ad-ineligible.
- `ads.txt` is emitted only when a valid publisher ID is configured via environment.

### Public information pages, header, and footer

- Substantial original marketing/policy content for all ten public routes in every
  locale, with responsive public header (logo, navigation, language dropdown, open-app
  CTA) and complete footer (product navigation, policies, repository link, version).

### Engineering and validation workflow

- `npm run i18n:check`, `npm run public:generate`, and `npm run public:validate` are
  wired into the Vercel preview validation pipeline (`scripts/validate-preview.mjs`).
- Rules and skills updated: 12-locale catalog requirements, truthful-layer placement for
  constants/types/interfaces/enums, hook-free components, and prerender validation steps.

## Known limitations

- Public marketing translations are professionally reviewed machine-assisted copy;
  native-speaker review remains a follow-up for `hi`, `th`, and `fa` long-form pages.
- AdSense remains disabled pending account review; manual ad units are not rendered
  anywhere yet.
